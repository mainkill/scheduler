프로젝트명: Scheduler 웹 앱 MVP
문서 버전: v0.1
작성일: 2026-03-20

1. 제품 개요
- 문제: 개인 사용자는 일정, 할 일, 마감일을 한 곳에서 빠르게 관리하기 어렵다.
- 해결책: 일정/할 일을 생성, 조회, 수정, 완료 처리할 수 있는 단순한 웹 앱을 제공한다.
- 핵심 가치: 빠른 입력, 명확한 우선순위, 날짜 기반 가시성.

2. 목표와 비목표
- 목표:
  - 개인 일정 관리의 핵심 흐름을 완성한다.
  - 일/주/월 일정 보기 기능을 지원한다.
  - 할 일 완료율과 마감 준수율을 높인다.
- 비목표:
  - 팀 협업 기능.
  - 고급 AI 자동 스케줄링.
  - 외부 캘린더 양방향 동기화.

3. 타깃 사용자
- 주요 사용자: 학생, 1인 개발자, 프리랜서, 일반 개인 사용자.
- 사용자 성향: 단순하고 빠른 UX를 선호하며 복잡한 설정을 피한다.

4. 핵심 사용자 시나리오
- 사용자는 1분 이내에 오늘 할 일을 추가할 수 있다.
- 사용자는 한 화면에서 이번 주 할 일을 확인할 수 있다.
- 사용자는 할 일을 완료 처리하고 상태를 빠르게 확인할 수 있다.
- 사용자는 기존 항목의 날짜/시간/우선순위를 변경할 수 있다.

5. MVP 범위
- Must Have:
  - 일정/할 일 항목 생성, 수정, 삭제.
  - 날짜와 시간 설정.
  - 우선순위 설정(높음, 중간, 낮음).
  - 일/주/월 보기 전환.
  - 완료/미완료 상태 토글.
  - 제목 기준 기본 검색.
- Should Have:
  - 태그 지원(최소 1개).
  - 완료 항목 숨김 필터.
- Could Have:
  - 반복 일정.
  - 알림 기능.
- 명시적 제외 범위:
  - 다중 사용자 협업.
  - 역할/권한 관리.
  - 외부 캘린더 연동(Google/Outlook).

6. 기능 요구사항
- FR-1: 사용자는 제목과 날짜로 할 일을 생성할 수 있어야 한다.
- FR-2: 사용자는 제목, 날짜/시간, 우선순위를 수정할 수 있어야 한다.
- FR-3: 사용자는 할 일을 삭제할 수 있어야 한다.
- FR-4: 사용자는 일/주/월 보기를 전환할 수 있어야 한다.
- FR-5: 사용자는 할 일을 완료/미완료로 표시할 수 있어야 한다.
- FR-6: 사용자는 제목 키워드로 할 일을 검색할 수 있어야 한다.

7. 비기능 요구사항
- NFR-1: 일반 네트워크 환경에서 메인 화면 최초 로드는 2초 이내여야 한다.
- NFR-2: 핵심 동작(생성/수정/완료)은 300ms 이내에 반응해야 한다.
- NFR-3: 데스크톱과 모바일 세로 화면에서 모두 사용 가능해야 한다.
- NFR-4: 기본 접근성(라벨, 키보드 포커스)을 지원해야 한다.

8. 데이터 모델(초안)
- Task
  - id: string
  - title: string
  - description: string (선택)
  - dueDate: datetime
  - priority: high | medium | low
  - status: todo | done
  - createdAt: datetime
  - updatedAt: datetime

9. 수용 기준
- AC-1: 새 할 일 생성 후 목록에 즉시 표시되어야 한다.
- AC-2: 수정한 값은 새로고침 후에도 유지되어야 한다.
- AC-3: 완료된 항목은 done 상태로 시각적으로 구분되어야 한다.
- AC-4: 일/주/월 보기에서 해당 기간 항목만 표시되어야 한다.
- AC-5: 검색 시 제목 키워드와 일치하는 항목만 필터링되어야 한다.

10. 성공 지표
- 첫 주 테스트에서 할 일 생성 성공률 90% 이상.
- 생성된 할 일 대비 완료 비율 40% 이상.
- 핵심 흐름 이탈률 20% 이하.
- 사용자 피드백 사용성 점수 4.0/5 이상.

11. 리스크와 대응
- 리스크: 범위 확대로 MVP 일정 지연.
  - 대응: 초기 릴리스에서 Must Have 범위를 고정한다.
- 리스크: 캘린더 뷰 구현 복잡도.
  - 대응: 단순 레이아웃과 핵심 상호작용을 우선한다.
- 리스크: 데이터 유실 우려.
  - 대응: 생성/수정 시 검증 후 즉시 저장한다.

12. 일정(초안)
- 1주차: UI 기본 구조, 데이터 모델, 생성/수정/삭제.
- 2주차: 보기 전환, 완료 토글, 검색.
- 3주차: 안정화, 테스트, MVP 릴리스.

13. 미결정 사항
- 로컬 알림 기능을 MVP에 포함할지 여부.
- 외부 연동 기능을 모두 Post-MVP로 미룰지 여부.
